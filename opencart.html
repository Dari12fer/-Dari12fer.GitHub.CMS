OpenCart

OpenCart — бесплатная платформа для создания коммерческих сайтов. Базовая функциональность платформы позволяет создать простой интернет-магазин. Несмотря на небольшую долю рынка, — около 3%, данная CMS всё ещё пользуется популярностью для создания онлайн-магазинов.

Сайты, для которых подойдёт OpenCart

Любые проекты, связанные с продажей товаров. У платформы очень ограниченный набор функций, поэтому её можно использовать только для сайтов в сфере e-commerce.

Удобство администрирования

Простая система добавления товара и работы с заявками не вызовет трудностей у большинства пользователей. Личный кабинет полностью русифицирован. Навыки программирования не требуются, но будет полезно получить элементарное представление о языке PHP — с его помощью можно внести изменения в исходный код.

Модули и утилиты

Для платформы с открытым исходным кодом было разработано более 13 000 утилит и модулей. С их помощью текстовый контент сайта можно легко перевести на десятки иностранных языков.

Модули автоматической конвертации валют и калькулятор скидок помогут работать с зарубежными покупателями. Можно настроить автоматический кэшбэк, карточку покупателя, создать воронку продаж.

Открытый исходный код позволяет создавать утилиты, скрипты, оформлять витрины товаров по своему вкусу.

Системные требования

  • Веб-сервер: IIS, Apache, Nginx
  • OS: UNIX, Windows
  • PHP 5.4 и выше
  • БД: MySQLi
  • Поддержка протокола HTTPS

Большинство платных хостингов поддерживает эту CMS и предлагает автоустановку одной из последних версий на русском или английском языке.

Безопасность и производительность

Система безопасности отслеживает действия пользователей и предоставляет возможность администратору блокировать пользователя или ограничивать доступ к выбранным функциям CMS.

Основные функции защиты:

  • защита от ботов и автоввода капчи
  • модерация контента и комментариев

Поддержка безопасности осуществляется с помощью SSL-сертификата.

Бесплатная платформа не рассчитана на крупные проекты, с ростом количества позиций могут появиться проблемы со скоростью загрузки. Временным решением может стать функция кэширования содержимого страниц. Загрузка дополнительных модулей может снизить нагрузку.

Для поддержки обширной базы данных и сотен покупателей ежедневно, потребуется качественный хостинг на SSD с высокой производительностью.

Возможности для самостоятельного конфигурирования

Базовой конфигурации достаточно для запуска интернет-магазина. По желанию пользователь может вносить изменения в ядро CMS, редактируя его с помощью языка PHP, написать джаваскрипт, добавить модуль. Виртуального редактора для работы с дизайном страниц нет. При работе с дополнениями рекомендуется учесть следующий факт: отдельные модули часто конфликтуют друг с другом, вызывая замедление загрузки страницы. Все дополнения необходимо проверять на совместимость.
